# Лаба 3. Очередь задач: итераторы и генераторы

## Запуск
```bash
uv venv
uv sync
source .venv/bin/activate
# для винды: .venv\Scripts\activate

# запуск
python -m src.main

# тестирование
pytest
```

## Принятые решения
- Очередь `Queue`. Пользовательская коллекция, работующая по принципу FIFO.
- Имеет методы `add`, `dequeue`, `extend`, `filter`, `clear`
- Имеет магические методы `__add__`, `__iter__`, `__getitem__`, `__len__`, `__eq__`
- `filter` возращает генератор. Можно фильтровать задачи по `priority` и `status`
- Интерактивый интерфейс реализован в классе `Cli`. `Cli` использует `Aggregator` чтобы получать и запускать задачи, и `TaskQueue` чтобы их хранить и фильтровать.
- Доступные команды: `help`, `aggregate`, `print`, `filter`, `run`, `clear`
- Задача `Task`. Класс с чистым публичным API, использующий декоратор `@property`
- Пример data descriptor: `priority` и `status`
- Пример non-data descriptor: `time_created`
- Контракт `Source`. Протокол который задает метод `get_tasks(): -> list`
- Класс `Aggregator`. Позволяет добавить источники, собирать задачи от этих источников и обрабатывать задачи.
- Проверяет соблюдение контракта источниками, создает задачи `Task`, дает им `id`, случайный приоритет, и обрабатывает их, при этом меня статус выполнения задачи
